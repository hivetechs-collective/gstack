## 2. SDK Breaking Changes (CRITICAL)

### System Prompt Change

**BREAKING**: The Claude Agent SDK **no longer uses Claude Code's system prompt
by default**.

**Impact**: SDK implementations that relied on Claude Code behavior need
explicit system prompts.

**Before (implicit)**:

```typescript
const result = query({
  prompt: "Review this code",
  options: { model: "claude-sonnet-4-6" },
  // System prompt was automatically included
});
```

**After (explicit required)**:

```typescript
const result = query({
  prompt: "Review this code",
  options: {
    model: "claude-sonnet-4-6",
    systemPrompt: {
      type: "text",
      text: "Your custom system prompt here",
    },
    // OR use appendSystemPrompt to add to Claude Code's default:
    // appendSystemPrompt: "Additional instructions"
  },
});
```

**Recommendation**: Use `settingSources: ['project']` to load project CLAUDE.md
files:

```typescript
const result = query({
  prompt: "Review this code",
  options: {
    model: "claude-sonnet-4-6",
    settingSources: ["project", "global"], // Loads CLAUDE.md files
  },
});
```

### Package Rename Complete

**Official rebrand** from Claude Code SDK → Claude Agent SDK:

```bash
# OLD (deprecated)
npm uninstall @anthropic-ai/claude-code

# NEW (current)
npm install @anthropic-ai/agent-sdk
```

**Import Changes**:

```typescript
// OLD

// NEW

```

## 3. New Claude Code Features

### Checkpoints & Rewind

Claude Code now automatically saves code state before making changes.

**Features**:

- Automatic checkpoint creation before file modifications
- Rewind to any previous checkpoint
- Undo risky operations safely

**Usage**:

- Press `Esc` twice during execution to interrupt
- Use `/rewind` command to restore previous state
- View checkpoints with `/checkpoints` command

**Agent Impact**: More aggressive refactoring safe - can always rewind

### VS Code Extension (Beta)

Native VS Code integration for Claude Code:

**Features**:

- Real-time code viewing in sidebar panel
- Inline diff visualization
- Seamless IDE integration

**Installation**:

- Available in VS Code marketplace (beta)
- Requires Claude Code CLI installed

### Subagents - Production Ready

Subagents are now **fully production-ready** with enhanced capabilities:

**Key Capabilities**:

- Isolated context windows (prevent context pollution)
- Custom system prompts (task-specific expertise)
- Configurable tool access (deny-all baseline with allowlists)
- Auto-detection from `.claude/agents/` directories
- Parallel execution (up to 10 concurrent, queue additional)

**Orchestrator-Worker Pattern** (2025 best practice):

```
Lead Agent (Orchestrator - usually Opus 4)
    ├─ Specialized Subagent 1 (Sonnet 4.6)
    ├─ Specialized Subagent 2 (Sonnet 4.6)
    ├─ Specialized Subagent 3 (Sonnet 4.6)
    └─ Lightweight Subagent N (Haiku 4.5)
```

### Background Tasks

Keep dev servers running without blocking other work:

```bash
# Start background task
claude --background "npm run dev"

# Continue working while server runs
claude "Fix the authentication bug"
```

### Structured Outputs (Public Beta)

Guaranteed schema conformance for API responses:

**Header Required**:

```

```

**Usage**: Ensures JSON output matches specified schema exactly

### MCP Connector (Public Beta)

Connect to remote MCP servers directly from Messages API:

- No need to run local MCP server
- Authentication handled automatically
- Useful for SaaS MCP providers

## 4. Hook System Updates

### Enhanced Hook Types

| Event                       | Trigger                      | Output                          |
| --------------------------- | ---------------------------- | ------------------------------- |
| **PreToolUse**              | Before tool execution        | Allow/deny/request permission   |
| **PostToolUse**             | After successful execution   | Modify results, trigger cleanup |
| **UserPromptSubmit**        | Before processing user input | Validate/modify prompts         |
| **Stop/SubagentStop**       | Agent finishes response      | Cleanup, notifications          |
| **SessionStart/SessionEnd** | Session lifecycle            | Setup/teardown logic            |
| **Notification**            | System alerts                | Custom handlers                 |

### MCP Tool Naming Pattern

```
mcp__<server_name>__<tool_name>

Examples:
mcp__memory__create_entities
mcp__filesystem__read_file
mcp__github__search_repositories
```

### Hook Security Best Practices

1. **Validate and sanitize all inputs**
2. **Always quote shell variables**: `"$VAR"` not `$VAR`
3. **Block path traversal**: Check for `..` in file paths
4. **Use absolute paths**: Specify full paths, use `$CLAUDE_PROJECT_DIR`
5. **Avoid sensitive files**: `.env`, `.git/`, private keys
6. **60-second default timeout** (configurable per command)

## 5. CLI Command Updates

### New/Updated Flags

```bash
claude                                      # Interactive session
claude -p "query"                          # Single query (headless)
claude -c                                  # Continue previous session
claude --system-prompt "custom"            # Replace full prompt
```

## 7. Workflow Best Practices (2025)

### "Explore, Plan, Code, Commit" Pattern

1. **Explore**: Have Claude read relevant files WITHOUT coding
2. **Plan**: Request plan with extended thinking (`--thinking`)
3. **Code**: Implement after plan approval
4. **Commit**: Automatic semantic commits after verification

### Test-Driven Development with Claude

```bash
# 1. Write tests first
claude -p "Write tests for user registration endpoint"

# 2. Confirm tests fail
npm test

# 3. Have Claude implement to pass tests
claude -p "Implement user registration to pass the tests at /path"

# 4. Verify
npm test  # Should pass now
```

### Visual Iteration Loop

```
1. Provide screenshot/design mock
2. Claude implements design
3. Screenshot results
4. Iterate on appearance (2-3 cycles typically)
```

### Parallel Agent Strategy

```bash
# Session 1: One instance writing code
claude -p "Implement feature A"

# Session 2 (parallel): Another instance reviewing
claude -p "Review feature A implementation"

# Key: Use git worktrees to prevent merge conflicts
```

### Model Selection Strategy

```typescript
// ALL CODING AND CRITICAL WORK → Opus 4.6 (MANDATORY)
const codeResult = await query({
  prompt: "Implement the authentication system",
  options: { model: "claude-opus-4-6" }, // Best coding model in the world
});

// MECHANICAL OPERATIONS ONLY → Haiku 4.5
const scanResult = await query({
  prompt: "List all TypeScript files in src/",

// DOCUMENTATION (optional cost savings) → Sonnet 4.6

  prompt: "Generate README for this module",
```

### Cost Comparison (per 1M tokens)

| Task      | Wrong Model     | Correct Model       | Savings                            |
| --------- | --------------- | ------------------- | ---------------------------------- |
| Coding    | Sonnet ($3/$15) | **Opus ($5/$25)**   | **Better results, worth the cost** |
| File scan | Opus ($5/$25)   | Haiku ($0.25/$1.25) | 95% savings                        |
| Docs      | Opus ($5/$25)   | Sonnet ($3/$15)     | 40% savings                        |

### Prompt Caching Best Practices

1. **Cache system prompts** (rarely change, 90% cost reduction)
2. **Cache tool definitions** (stable across conversations)
3. **Cache conversation history** (diminishing returns after ~10 messages)
4. **Place cache_control on LAST item** in array (Anthropic requirement)
5. **Minimum 1024 tokens** for cache efficiency
