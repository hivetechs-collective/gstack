## Key Findings

### 1. Agent Metadata `color` Field Resolution

**Question**: Is it `color` or `x-color`?

**Answer**: **Use `color`** (not `x-color`)

**Evidence**:

- GitHub Issue #8501 confirms `color` field exists but is undocumented
- `/agents` command in Claude Code generates agents with `color: <name>` field
- Community examples show `color: purple`, `color: yellow`, etc.
- No evidence of `x-color` being functional or preferred

**Important Caveat**:

- Colored badges **removed in Claude Code v2.0.11** (Oct 8, 2025)
- Field still accepted but not visually displayed
- GitHub Issue #9319 reports this regression (closed as duplicate)

### 2. Official Documentation Gaps

**Confirmed Documentation Issues**:

- GitHub Issue #8501 (OPEN since September 2025)
- Official docs at docs.claude.com do NOT mention `color` field
- Official docs do NOT provide complete frontmatter specification
- `/agents` command generates fields not in documentation

**Community Impact**:

- Developers rely on implementation behavior vs documentation
- Community tools create extended metadata fields
- No authoritative reference for agent configuration

### 3. Claude Code Version Tracking

**Current Version**: 2.0.29 (October 2025)

**Recent Changes**:

- v2.0.10 (Oct 7): Colored badges working ✅
- v2.0.11 (Oct 8): Badges removed (system prompt optimization) ⚠️
- v2.0.12-2.0.29: Badges still missing ⚠️

**Known Issues**:

- Visual regression in agent identification
- No official timeline for restoration

### 4. settings.json Configuration

**Finding**: No `agentDescriptions` field exists

**Clarification**:

- Agents are configured via `.claude/agents/*.md` files
- `settings.json` controls permissions, hooks, environment
- Agent metadata is in YAML frontmatter, not settings.json

**Available settings.json Options**:

- `allow`/`deny`/`ask` - Permission controls
- `env` - Environment variables
- `model` - Model selection
- `hooks` - Lifecycle hooks
- `statusLine` - Status display
- `settingSources` - Filesystem settings loading

## Breaking Changes & Important Updates

### SDK Package Rename (BREAKING)

**Old Package Names**:

- TypeScript: `@anthropic-ai/claude-code`
- Python: `claude-code-sdk`

**New Package Names**:

- TypeScript: `@anthropic-ai/claude-agent-sdk`
- Python: `claude-agent-sdk`

**Migration Required**: Update imports and package.json/requirements.txt

### System Prompt Behavior Change (BREAKING)

**Change**: SDK no longer uses Claude Code's system prompt by default

**Impact**: Agents may behave differently without explicit system prompt

**Fix**:

```typescript
options: {
  systemPrompt: { type: "preset", preset: "claude_code" }
}
```

### Settings Loading Change (BREAKING)

**Change**: Filesystem settings no longer loaded automatically

**Impact**: Skills, subagents, hooks not available by default

**Fix**:

```typescript
options: {
  settingSources: ['user', 'project', 'local'];
}
```
